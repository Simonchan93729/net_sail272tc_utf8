## 🚀 上傳方式與權限設定
### 伺服器目錄結構（範例）
          cgi-bin
            └ sea(777 or 707)
                ├ sea.cgi(755)
                ├ new.cgi(755)
                ├ list.cgi(755)
                ├ admin.cgi(755)
                ├ 其他的.cgi(644)
                ├ index.html
                ├ img資料夾
                │  └ 船艦圖像
				│
                ├ usrdir資料夾(777 or 707)(資料夾名稱一定要更改！)
                ├ city資料夾(777 or 707)(資料夾名稱一定要更改！)
                ├ data資料夾(777 or 707)(資料夾名稱一定要更改！)
                │   └ dat資料內的(1001～1009.dat、admin.dat皆設定666，其他全設644)
				│
				├ bbs資料夾(777 or 707)
				│　 ├ say.cgi(755)
				│　 ├ cgi-lib.pl(644)
				│　 └ log.dat(666)
				│
				└ event.dat(666)

### 權限設定重點
- 可寫入資料夾（usrdir/city/data/bbs）：權限設為 `777` 或 `707`；
- 核心 CGI 執行檔（sea.cgi/new.cgi 等）：權限設為 `755`；
- 可寫入數據檔（1001～1009.dat/admin.dat/log.dat）：權限設為 `666`；
- 其餘文件（說明檔、圖片等）：權限設為 `644`。

## 自由擴充教學
- 遊戲中的「交易品」、「船艦」、「冒險情報」均可自行增加、刪除

### 追加新冒險情報
- 我們先做增加冒險任務的教學
- 舉例：ostrich.dat
- 1004,052,4,「關於那個腳很長大鳥的事情嗎？<br>看到後也許會讓你嚇一跳的。」
- 1004,052,6,發現了駝鳥,駝鳥的蛋

- 各行的意思說明：
- 1004,052,		代表著城市的編號，請參閱(1001～1009.dat）
- 1004,052,"4",		帶表城鎮內的設施
- 『海上＝0、港口＝1、交易所＝2、造船場＝3、酒場＝4、城鎮＝5、襲擊＝6』
- 另外..最後一行「1004,052,6,發現了駝鳥,駝鳥的蛋」是任務最終的結尾
- 句子最後的「鴕鳥的蛋」為寶物

- 整體來說，任務的架構就是
- 『海域代碼,城市代碼,設施代碼,情報訊息,寶物』
- 寶物只有寫在冒險情報最後一段才有作用(可加可不加)

- 當編輯完之後，請存檔成*.dat
- 請自行摸索~

### 追加新冒險情報酒館列表
- 上面只是冒險情報的寫法，還不會出現在酒館的冒險情報內
- 這時你必須編輯：a_list.dat

- 舉例：
- <>001<>400<>fsta.dat<>「交易品店的老闆總覺得知道某些事情。」<>
- 1001<><>250<>pisa.dat<>「去地中海可以看見比薩斜塔。」<>

- 解說：
- <>001<>400<>fsta.dat<>「交易品店的老闆總覺得知道某些事情。」<>
- <>城市代碼<>價格<>冒險情報.dat<>初始情報訊息<>

- 1001<><>250<>pisa.dat<>「去地中海可以看見比薩斜塔。」<>
- 海域代碼<><>價格<>冒險情報.dat<>初始情報訊息<>

- Q:上面這兩段有什麼不同
- A:第一段式獨立任務，只有指定的城市才購買得到
-   第二段則是共同任務，同一海域均可購買得到

- 請自行摸索~

### 追加新城市、新貨物
- 001<>里斯本<>38,-9<>岩鹽,30△橄欖油,100△洋槍,200△大砲,300△銀,280△金,550△古董品,300<>1<>

- 城市代碼<>城市名稱<>北緯(南緯),東經(西經)<>物品名1,價格1△物品名2,價格2（中間須以"△"隔開）<>物價(1=100%)<>

- 另外，base.dat為貨物基本價格，在追加新貨品之後必須加以編輯增加

### 追加新船艦
- 編輯：item.dat
- 輕木帆船<>1<>balsha.gif<>1001<><>50<>80<>7.5<>1200<>1<>2<>

- 名字<>船艦均是1<>圖像<>販賣的海域(註1)<>販賣的城市(註2)<>積載量<>耐久<>速度<>價格<>
- (註1、2，請對照上面「追加新冒險情報酒館列表」的解說)
